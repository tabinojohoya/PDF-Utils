# PDF-Utils v1.1.0 Release Notes

## ページ単位編集 & 出力プレビュー

PDF-Utils v1.1.0 では、ファイル単位の結合に加え、**ページ単位での編集**と**結合結果の即時プレビュー**を追加しました。

---

### ✨ New Features

#### ページビュー
- ツールバーのファイル/ページ切替ボタン、または `⌘1` / `⌘2` で表示モードを切り替え可能に
- 各PDFをファイルセクションごとに展開し、全ページのサムネイルをグリッド表示（80×104pt）
- セクションは折りたたみ可能（DisclosureGroup）

#### ページ単位の除外・選択
- 各ページのチェックボックスで個別に結合対象から除外可能
- 除外ページは半透明（opacity 0.35）+ 目のスラッシュアイコンで視覚的に区別
- ファイルヘッダの全選択/全解除トグルで一括操作
- 全ページ除外されたファイルがある場合、ステータスバーに警告を表示
- 結合対象ページが0件の場合、結合ボタンを自動的に無効化

#### ページ並び替え
- ファイル内でページをドラッグ&ドロップで並び替え可能

#### プレビュー連動
- ページサムネイルをクリックすると、右ペインに該当ファイルのPDFプレビューを表示

#### 出力プレビュー
- 結合完了後、右ペインが自動的に結合済みPDFのプレビューに切り替わる
- 連続スクロールモードで全ページを確認可能
- 「← 編集に戻る」ボタンで元のファイル/ページ選択画面に復帰
- ソースファイルを変更（追加・削除・ページ編集・並び替え）すると自動的に編集画面に戻る

#### ステータスバーの強化
- ページ除外時: `3ファイル · 25/37ページ選択中` 形式で表示
- 出力プレビュー中: `3ファイル · 結合結果: 34ページ · 4.2 MB` 形式で表示

---

### 🐛 Bug Fixes

- ファイル並び替え（移動）後に出力プレビューが解除されない問題を修正
- ステータスバーが毎レンダリングでPDFファイルを再読み込みする性能問題を修正（結合結果情報をキャッシュ化）
- 全ページ削除済みファイル（pages配列が空）で0ページ警告が誤表示される問題を修正
- ページサムネイルのチェックボックスとタップジェスチャの競合を修正

---

### 📁 Added Files

- `Models/PageItem.swift` — ページ単位のデータモデル
- `Views/PageGridView.swift` — ページビュー（サムネイルグリッド + ファイルセクション）
- `Views/PageThumbnailView.swift` — ページサムネイル1枚のコンポーネント
- `Views/MergedPreviewView.swift` — 結合結果の出力プレビュー

### 📝 Modified Files

- `Models/PDFItem.swift` — `pages: [PageItem]` プロパティ追加
- `ViewModels/PDFMergeViewModel.swift` — ページ操作・表示モード・出力プレビュー状態管理を追加
- `Services/PDFMergeService.swift` — ページ単位の結合に対応（`pageIndices` 指定）
- `ContentView.swift` — ファイル/ページ表示切替、出力プレビュー切替、ステータスバー強化

---

### ⌨️ Keyboard Shortcuts

| Shortcut | Action |
|----------|--------|
| `⌘1` | ファイルビューに切替 |
| `⌘2` | ページビューに切替 |

---

### ⚠️ Known Limitations

- ページのファイル間ドラッグ&ドロップ（ファイルをまたいだ並び替え）は未対応
- パスワード保護PDFは引き続き読み込み不可
