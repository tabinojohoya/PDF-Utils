# PDF-Utils 要件定義書

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| アプリ名 | PDF-Utils |
| プラットフォーム | macOS Tahoe (26.x) |
| 開発言語 | Swift 5 (Swift 6 Concurrency対応) |
| UIフレームワーク | SwiftUI |
| 開発環境 | Xcode 26.2 |
| バンドルID | com.example.PDF-Utils |
| 最小動作OS | macOS 26.2 |

## 2. アプリの目的

複数のPDFファイルを選択し、指定した順序で1つのPDFファイルに結合するmacOSネイティブアプリケーション。

## 3. 機能要件

### 3.1 PDF ファイル追加 (FR-001)

| 項目 | 内容 |
|------|------|
| 概要 | ユーザーが結合対象のPDFファイルを追加できる |
| 入力方法 | ファイル選択ダイアログ（複数選択対応） |
| 入力方法 | ドラッグ&ドロップ（複数ファイル対応） |
| 対象形式 | `.pdf` ファイルのみ |
| バリデーション | PDF形式でないファイルは拒否し、エラーメッセージを表示 |

### 3.2 PDF ファイル一覧表示 (FR-002)

| 項目 | 内容 |
|------|------|
| 概要 | 追加されたPDFファイルをリスト形式で表示 |
| 表示項目 | ファイル名、ページ数、ファイルサイズ |
| サムネイル | 各PDFの先頭ページのサムネイルを表示 |

### 3.3 並び替え (FR-003)

| 項目 | 内容 |
|------|------|
| 概要 | 結合順序をユーザーが自由に変更できる |
| 操作方法 | ドラッグ&ドロップによるリスト内並び替え |
| 操作方法 | 「上へ移動」「下へ移動」ボタン |

### 3.4 ファイル削除 (FR-004)

| 項目 | 内容 |
|------|------|
| 概要 | リストから個別のPDFファイルを除外できる |
| 操作方法 | 選択してDeleteキーまたは削除ボタン |
| 一括操作 | 「すべてクリア」ボタンで全件削除 |

### 3.5 PDF プレビュー (FR-005)

| 項目 | 内容 |
|------|------|
| 概要 | 選択したPDFの内容をプレビュー表示 |
| 表示方法 | 右側ペインにページプレビューを表示 |
| ページ送り | 複数ページのPDFはページ送りが可能 |

### 3.6 PDF 結合実行 (FR-006)

| 項目 | 内容 |
|------|------|
| 概要 | リスト内の全PDFを上から順に結合し、1つのPDFとして出力 |
| 保存方法 | NSSavePanelによる保存先選択ダイアログ |
| デフォルト名 | `Merged.pdf` |
| 処理表示 | 結合中はプログレスバーを表示 |
| 完了通知 | 結合完了後に成功メッセージを表示 |
| エラー処理 | 失敗時はエラー内容を表示 |

### 3.7 結合結果プレビュー (FR-007)

| 項目 | 内容 |
|------|------|
| 概要 | 結合完了後、結果のPDFをQuick Lookまたはプレビューで確認できる |
| 操作 | 「結合結果を開く」ボタンで結合済みPDFを開く |

## 4. 非機能要件

### 4.1 パフォーマンス (NFR-001)

- 100ページ以上のPDFでも数秒以内に結合処理を完了すること
- PDF結合処理はバックグラウンドスレッドで実行し、UIをブロックしないこと
- メモリ効率を考慮し、大容量PDFでもメモリ不足にならない設計とすること

### 4.2 セキュリティ (NFR-002)

- App Sandbox有効（macOS標準のセキュリティモデルに準拠）
- ユーザーが明示的に選択したファイルのみアクセス（`ENABLE_USER_SELECTED_FILES`）
- Hardened Runtime有効

### 4.3 ユーザビリティ (NFR-003)

- macOSネイティブのルック&フィールに準拠
- キーボードショートカット対応（⌘O: ファイル追加、⌘S: 結合実行、⌘⌫: 選択削除）
- ダークモード / ライトモード両対応

### 4.4 信頼性 (NFR-004)

- 破損したPDFファイルについて適切なエラーハンドリングを行うこと
- パスワード付きPDFはスキップし、ユーザーに通知すること

## 6. GUI設計思想

> *"Design is not just what it looks like and feels like. Design is how it works."*
> — Steve Jobs

本アプリのGUIは「**引き算の美学**」に基づく。機能を足すのではなく、ユーザーが考えなくて済むものを一つずつ取り除く。PDFを結合する——たったそれだけのことに、マニュアルは要らない。アプリを初めて開いた瞬間に、次に何をすべきかが分かる。それが正しいデザインである。

### 6.1 デザイン原則

| # | 原則 | 具体的な指針 |
|---|------|-------------|
| 1 | **極限のシンプルさ** | 画面上の全要素に存在理由を問う。説明できないものは削除する |
| 2 | **一目で伝わる階層** | 最も重要なアクション（結合実行）が視覚的に最も目立つこと |
| 3 | **直接操作の快感** | ドラッグ&ドロップは「掴んで、置く」。人間が物を扱う自然な動作そのもの |
| 4 | **待たせない、退屈させない** | 全てのインタラクションに即座のフィードバック。沈黙は最悪のUI |
| 5 | **一貫性という信頼** | macOSネイティブの作法に従う。独自UIは作らない。Appleが正しい |
| 6 | **余白は贅沢ではなく必然** | 詰め込まない。呼吸できる空間がコンテンツを引き立てる |

### 6.2 カラーシステム

独自カラーは定義しない。macOSのセマンティックカラーに100%準拠する。

| 用途 | カラー指定 | 理由 |
|------|-----------|------|
| 背景 | `.background` (System) | OSのモード切替に自動追従 |
| テキスト（主） | `.primary` | 可読性最優先 |
| テキスト（副） | `.secondary` | 情報の階層を自然に伝える |
| アクセント | `.accentColor` (System Blue) | OSの設定をユーザーが変更可能 |
| 結合ボタン | `.tint` + `.borderedProminent` | 画面内で唯一の「主張する色」 |
| ドロップゾーン（アクティブ） | `.blue.opacity(0.08)` + 破線ボーダー | 受け入れ可能状態を穏やかに示す |
| エラー | `.red` (System) | 赤は「注意」の万国共通語 |
| 成功 | `.green` (System) | 完了の安心感 |
| サムネイル背景 | `.quaternary` | コンテンツを邪魔しない最小限の境界 |

**ダークモード**: 全てのセマンティックカラーがOSレベルで自動反転する。カスタム対応コードはゼロ。

### 6.3 タイポグラフィ

システムフォント（San Francisco）のみ使用。カスタムフォントは不要。

| 要素 | スタイル | サイズ区分 |
|------|---------|-----------|
| ウィンドウタイトル | OS標準タイトルバー | — |
| ツールバーラベル | `.body` | 13pt |
| ファイル名 | `.body` `.fontWeight(.medium)` | 13pt |
| ページ数・サイズ | `.caption` `.foregroundStyle(.secondary)` | 10pt |
| 空状態メッセージ（主） | `.title2` `.fontWeight(.medium)` | 17pt |
| 空状態メッセージ（副） | `.subheadline` `.foregroundStyle(.secondary)` | 12pt |
| ステータスバー | `.caption` `.foregroundStyle(.secondary)` | 10pt |
| 結合ボタン | `.body` `.fontWeight(.semibold)` | 13pt |

### 6.4 アイコノグラフィ

SF Symbolsのみ使用。サードパーティアイコンは一切不要。

| 用途 | SF Symbol名 | レンダリング | 理由 |
|------|------------|-------------|------|
| ファイル追加 | `plus` | `.monochrome` | 最も普遍的な「追加」の記号 |
| 削除 | `trash` | `.monochrome` | macOS標準の削除メタファー |
| 全クリア | `trash.slash` | `.monochrome` | 「全部消す」のニュアンス |
| 結合実行 | `doc.on.doc.fill` | `.hierarchical` | 2つの文書→1つへの暗示 |
| 上へ移動 | `chevron.up` | `.monochrome` | 方向の直感 |
| 下へ移動 | `chevron.down` | `.monochrome` | 方向の直感 |
| 空状態 | `doc.richtext` | `.hierarchical` | PDFの象徴。大きく表示 |
| ドラッグヒント | `arrow.down.doc` | `.hierarchical` | 「ここに落とせ」の視覚的合図 |
| エラー | `exclamationmark.triangle.fill` | `.multicolor` | 目を引くが威圧しない |
| 成功 | `checkmark.circle.fill` | `.multicolor` | 達成感のシグナル |
| ページ送り（前） | `chevron.left` | `.monochrome` | — |
| ページ送り（次） | `chevron.right` | `.monochrome` | — |

**レイアウトの鉄則**:
- ツールバーの左側は「素材を扱う操作」（追加・削除）、右側は「成果を生む操作」（結合）。この左→右の流れはワークフローそのもの
- 結合ボタンだけが `.borderedProminent` で、画面全体の中で唯一の「ここを押せ」ポイント
- サイドバーとプレビューの比率は約 **1:2.5**。プレビューが主役であり、リストは脇役

> *"成功は祝え。でも大げさにするな。"*

モーダルダイアログは使わない。画面上部にトースト風バナーをスライドイン表示。

```
┌─────────────────────────────────────────────────┐
│  ✅  結合が完了しました   [ Finderで表示 ]  [ ✕ ] │

```

- 5秒後に自動消滅（`.animation(.easeOut)`）
- 「Finderで表示」ボタンで保存先を即座に確認可能
- ユーザーの作業フローを邪魔しない

### 6.6 アニメーション設計

> *"アニメーションは装飾ではない。因果関係を伝える言語だ。"*

| トリガー | アニメーション | Duration | Curve |
|---------|-------------|----------|-------|
| ファイル追加 | リストにスライドイン | 0.3s | `.spring(response: 0.3, dampingFraction: 0.8)` |
| ファイル削除 | リストからフェード+縮小して消滅 | 0.25s | `.easeOut` |
| 並び替え | 行が移動先へスムーズに移動 | 0.3s | `.spring(response: 0.3, dampingFraction: 0.7)` |
| ドラッグ進入 | ドロップゾーンが穏やかに発光 | 0.2s | `.easeInOut` |
| ドラッグ退出 | ドロップゾーンがフェードアウト | 0.15s | `.easeOut` |
| プレビュー切替 | クロスフェード | 0.2s | `.easeInOut` |
| 結合開始 | プログレスバーがスライドイン | 0.3s | `.easeOut` |
| 結合完了 | 成功バナーが上からスライドイン | 0.4s | `.spring(response: 0.4, dampingFraction: 0.75)` |
| 成功バナー消滅 | フェードアウト | 0.5s | `.easeOut`（5秒後に発火） |
| 空状態→リスト表示切替 | 空状態がフェードアウト→SplitViewが展開 | 0.35s | `.easeInOut` |

**アニメーションの鉄則**:
- 0.5秒を超えるアニメーションは存在しない。ユーザーを待たせる演出は傲慢
- 全てのアニメーションに物理的な意味がある（上から落ちる、横からスライドする など）
- `withAnimation {}` で明示的にトリガーし、意図しない箇所が動かないようにする

### 6.7 インタラクションの状態遷移

```
                    ┌─────────────┐
                    │   空状態     │
                    │ (ゼロステート) │
                    └──────┬──────┘
                           │ ファイル追加（D&D or ダイアログ）
                           ▼
                    ┌─────────────┐
            ┌──────▶│  編集状態    │◀────── 並び替え
            │       │ (ファイル有) │◀────── 追加
            │       └──────┬──────┘─────▶ 削除（残0件で空状態へ）
            │              │
            │              │ 結合ボタン押下
            │              ▼
            │       ┌─────────────┐
            │       │  結合処理中   │  ← UIロック（追加・削除・並替え無効）
            │       │              │  ← プログレスバー表示
            │       └──────┬──────┘
            │              │
            │         ┌────┴────┐
            │         ▼         ▼
            │  ┌──────────┐ ┌──────────┐
            │  │  成功     │ │  失敗     │
            │  │ バナー表示 │ │ エラー表示 │
            │  └─────┬────┘ └─────┬────┘
            │        │            │
            └────────┴────────────┘

```

### Step 4: PDF結合サービス — PDFMergeService の実装

**目的**: PDF結合のコアロジックを分離した純粋なサービスとして実装

**作成ファイル**: `Services/PDFMergeService.swift`

**実装内容**:
```swift
struct PDFMergeService {
    /// 複数のPDFファイルを結合し、指定パスに保存する
    /// - Parameters:
    ///   - urls: 結合するPDFファイルのURL配列（順序通り）
    ///   - outputURL: 保存先URL
    ///   - progress: 進捗コールバック (0.0〜1.0)
    /// - Throws: PDFMergeError
    static func merge(
        urls: [URL],
        to outputURL: URL,
        progress: @Sendable (Double) -> Void
    ) async throws
}

enum PDFMergeError: LocalizedError {
    case fileNotReadable(URL)
    case passwordProtected(URL)
    case writeFailed(URL)
    case noFiles
}
```

**結合アルゴリズム**:
1. 出力用 `PDFDocument()` を生成
2. 各URLを順にループ:
   a. `PDFDocument(url:)` で読み込み（失敗→エラー）
   b. `isLocked` チェック（ロック→スキップ+通知）
   c. 全ページを `insert(_:at:)` で追記
   d. 進捗を更新
3. `write(to:)` で保存

**完了条件**: ビルド成功。ユニットテスト用にサービス単体でテスト可能な設計。

### Step 10: PDF結合機能の統合

**目的**: ViewModelと結合サービスを接続し、結合ワークフロー全体を完成させる

### Step 11: エラーハンドリングとエッジケース

**目的**: 堅牢性の確保

**対応項目**:

| # | ケース | 対応 |
|---|--------|------|
| 1 | 非PDFファイルのD&D | `.dropDestination` で `UTType.pdf` のみ受理。無視される |
| 2 | 破損PDFの追加 | `PDFDocument(url:)` が `nil` → エラーAlert表示、リストに追加しない |
| 3 | パスワード付きPDF | `isLocked == true` → Alert表示「{ファイル名}はパスワードで保護されています」 |
| 4 | 同一ファイルの重複追加 | URLの重複チェック → 既に追加済みの場合はスキップ+通知 |
| 5 | 結合対象が1件のみ | 結合ボタンを無効化（`canMerge = items.count >= 2`） |
| 6 | 結合中のUI操作 | `isMerging` 中はツールバーボタンを `.disabled(true)` |
| 7 | 保存先の書き込み権限なし | `PDFMergeError.writeFailed` → エラーAlert |

**変更ファイル**:
- `ViewModels/PDFMergeViewModel.swift`
- `Models/PDFItem.swift`（ファクトリメソッドにバリデーション）
- `Views/ContentView.swift`（`.alert` 表示の追加）

**完了条件**: 上記全ケースで適切なエラーメッセージが表示され、アプリがクラッシュしない。
